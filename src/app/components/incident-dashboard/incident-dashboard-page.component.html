Resilient Incident Dashboard

Power BI report with insights on incident distribution by location and ratings.


(click on the image to open in a new tab)


Data Sources

This project uses data from Resilient (list of incidents and types)).

Data Refresh

Incident reports are refreshed daily 05:00 AM UTC.

Model

The model has two tables: Incident: List of incidents with key fields. Incident Detail: List of incidents with all fields, with access allowed only to members of groups "G AAD ITANDS DS SDS RID Power BI APP Details" or "G AAD ITANDS DS SDS RID Power BI Workspace".

Metrics

The model uses the Discovered Date as the main date field. As some incidents have multiple values for some fields (Incident Type, Action Taken, Member, Isp Attach Vector, Isp Operating System, Impacted Business / Entity, KillChain Stage, Impacted Region, Source of Intelligence and I2p Software Download Infection), all counts are using a DISTINCT in the Incident ID.

Reports

Two reports are available from the Power BI Workspace "Incident Dashboard":

Incident Dashboard: This is the main report with pages for Overview, Location, Rating, Totals and Incident Details. The report has a filter to list only incidents with rating as P1, P2, P3 or P4.

Monitoring: Monitoring of the process to load new data into the model.
