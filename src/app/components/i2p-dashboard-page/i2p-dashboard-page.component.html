I2P

I2P is a Power BI Workspace with reports on incidents per person.


(click on the image to open in a new tab)


Data Sources

This project uses data from Resilient (list of incidents and types) and from Saviynt (list of users and their location).

Data Refresh

I2P reports are refreshed daily 05:00 AM UTC.

Model

The model has the following tables:

  • Incident: List of incidents with key fields.
  • Incident Detail: List of incidents with all fields.
  • Population: Count of population by Entity, Region and Country.
  • Saviynt: List of users used to calculate the population.

Metrics

The key measures of the Power BI model are:

  • Incident Count: distinct count of incident IDs
  • Incident %: Percentage of the incidents of a region/country/entity compared to the total incidents
  • Population Total: total of active users under a region/country/entity
  • Population %: Percentage of the population of a region/country/entity compared to the total population
  • I2P: Incident % divided by Population %

Reports

I2P Analysis: This is the main report with tabs for I2P, Attack Vector and Incident Details.

I2P Late Incidents: List of incidents that were created late.

Monitoring and Data Issues: Monitoring of the process to load new data and maintenance of configuration tables.
