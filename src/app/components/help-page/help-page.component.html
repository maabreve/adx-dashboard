Overview

This application is a data search and analytics tool that has been designed to allow data to be queried using multiple criteria that can target multiple data feeds over a specific period. The SDS ADXsearch application is an intuitive web interface, that allows users to select feeds, fields and drill-down to explore, summarise and export data without having to write ADX queries. This improves the capacity to filter and analyse data fields available from multiple data feeds.

ADXsearch

The ADXsearch application allows users to make a quick data search over the past 2 year’s worth of data using Azure Data Explorer.

What is Azure Data Explorer?

A big data analytics cloud platform optimized for interactive, ad-hoc queries over structured, semi-structured and unstructured data.

  1. Fully managed – We have no servers to manage or Patch!
  2. Ability to work with any kind of data: structured, semi-structured (JSON and more) and unstructured (free text).
  3. High velocity (millions of events per second), low-latency (seconds) and linear scale ingestion of raw data.
  4. Scalable query performance. (Auto Scaling)
  5. Rich and powerful query capabilities, supporting the whole complexity spectrum: from a simple keyword search to the complex time series and behavioural analytics.
  6. Simple, yet powerful and productive query language and toolset.
  7. Kusto – ‘KQL’ – used across all Microsoft tools particularly the hunting tools.

For more information on Azure Data Explorer please refer to Microsoft’s documentation.

Please also refer to Microsoft’s writing queries guide and kql quick reference to learn more about Kusto Query Language. Bp also provides a great training course on the learning portal called Kusto Query Language (KQL) from Scratch.

Feeds

The application can search across one or more of the following 9 data feeds:

  • Active Directory
  • VPNData
  • DefenderATP
  • DefenderAV
  • Falcon
  • IPData
  • McAfee ePolicy
  • Proxy Logs (IronPort, ScanSafe, Zscaler)
  • SourceFire

Within each of the feeds even more specific queries can be defined by selecting specific fields such as Source IP or Destination Host. Additionally, ADXsearch provides the capability to search the certain fields across all data feeds using tags which select the relevant fields under all the individual feeds, providing more seamless querying for data analysis.

How To

Building a new query

  1. Select one or more fields from the available Feeds by clicking on specific field names or tags, for example Name under the Active Directory feed and the all NTIDs tag across all 9 data feeds.
    When searching a certain data source field, if it has a target symbol next to it such as Name in the Active Directory example an exact match is used when querying the data source, for a result to be returned the whole term must be entered.
    Whereas the other fields such as Destination Host under the VPNData data feed will query using partial matches of the entered term and therefore can still get a result.
  2. Type and confirm one or more text filters – multiple terms can be pasted in one go as text separated by breaklines from a clipboard.
  3. Select the data range using either the following quick button options “Last Day”, “Last Week”, “Last Month”, “3 Months”, “6 Months” and “12 Months” or by picking specific start and end dates using the datetime picker – if no range is specified the default “Last Month” is used.
  4. Select the results limit (100, 1K, 10K[default], 100k).
  5. Click “Search” to proceed with the query configured or click “Reset” to clear selections.
  6. Selections are translated into an ADX query, which searches across the selected fields – the kusto query used is visible by clicking on the purple information icon top right on the result table.
  7. Whilst the query is running the application displays the spinning wheel and sends the fully query to ADX, there is also a “Cancel Query” button which can be clicked to terminate the query if it is taking too long.
  8. Results are shown as a table for each of the data feeds in seperate tabs in the main result area.
  9. A time clustered column chart can be viewed by clicking on the “Show Timeline” button in the bottom right by the corner of the results table.
  10. A clustered bar chart with the distribution of records by table (Proxy Logs, VPNData, etc.) can be viewed by clicking on the "Show Histogram” button in the bottom right by the corner of the results table.
  11. Once the results are loaded, additional filters can be added and logic operator (“AND”, “OR”, “AND NOT”) can be used.

Refining a query

User can click on the components of the result area to add additional filter to the data already received from ADX.

  1. Double-click on the timeline or histogram.
  2. The selected field and field value added are applied as a filter on the data already loaded from ADX.
  3. An indication of the additional filter is on the top of the main content area, so it this additional filter can be removed.

Exporting

  1. Search results can be export as CSV or Excel by clicking either the blue “Export to CSV” or green “Export to XLS” buttons on the top right of the table.
  2. File is generated and automatically downloaded.

SOCtopus

NTID

The NTID tool is used to investigate users either by their NTID or UPN.

At the top of the page there are some filters which can be used to narrow down the search after entering a NTID or UPN into the search bar on the left. The date range can be selected in a two ways, the first is by selecting one of the following the options “Last Day”, “Last Week”, “Last Month”, “3 Months”, “6 Months” and “12 Months” and the second is picking specific start and end dates using the datetime picker on the far right – if no range is specified the default “Last Month” is used.

The “User Information” section is populated with all the information pertaining to the NTID or UPN searched.

The information displayed for a user are:

  • Password Last Changed – the date and time (displayed as DD-MM-YY HH:MM:SS) when the password of the User was last modified.
  • Account Disabled – A yes or no answer showing if the user is still active.
  • NTID – the User’s NTID.
  • Most Recent Host – the most recent Host the User used to log into BP accounts.
  • Most Recent IP – the User’s most recent IP.
  • Address – the address of the office the User is located.
  • Email – the User’s email address, this is provided as a hyper link that redirects the searching user to their email address and opens a new email so that the searching user may immediately send an email to the searched User.
  • Phone – the User’s phone number, displayed with an area code for ease of contact. This is also a hyperlink, which allows the searching user to make a call to the searched User.
  • Job Title – the job title of the User.
  • Company – the specific company the User works for.
  • Region Code – the region code of where the User works.
  • Site Code – the site code of where the User works.
  • Entity – the entity of the business the User works for.
  • Sub-Entity – the sub-entity of the business the User works for.
  • Enterprise – the enterprise of the business the user works for.
  • Manager – the User’s line manager.
  • Manager NTID – the User’s line manager’s NTID.

The user information can be easily copied to clipboard by clicking the copy button in the top right of the box.

The next section has two tabs:

1) Alerts

The Alerts tab shows any data pertaining to the User where any action they have taken has triggered an alert. The data displayed is in two columns, and for ease of understanding, the data displayed is headed by the data source the information comes from.

  1. Advanced Threat Detections
    • On the left side, a box labelled “Crowdstrike Falcon” displays the following data:
      • Adware Detection
      • Blocked Hash Detection
      • Command Detection
      • Credential Theft Detection
      • CS Intel Detection
      • DLL Detection
      • File Detection
      • ISS Detection
      • Malware Detection
      • Miner Detection
      • Network Detection
      • Overwatch Detection
      • Process Detection
      • Ransomware Detection
      • Other
    • On the right side, the following data is displayed:
      • Windows Defender ATP Alerts
      • Fireeye Events
      • SourceFire Events
  2. Antivirus Detections
    • On the left side, the following data is displayed:
      • Falcon NetGen AV Alerts
      • Windows Defender ATP Alerts
    • On the right side, the following data is displayed:
      • McAfee ePO Events

At the end of each row a number is displayed for each individual search result to show how many incidents of each category have triggered an alert. Each number is a hyperlink, which redirect to the ADXsearch tool and to the specific search result, to facilitate further investigation of the data results.

2) AD Events, Hosts & IPs

This tab has 4 main boxes which display the following:

  1. Top AD Events – lists the top 5 AD Events, along with their relevant code and the number of occurrences per event type.
  2. Top 5 IPs – lists the top 5 IPs the user has accessed and the number of occurrences.
  3. The Top 5 Host – lists the top 5 hosts the user has used and the number of occurrences.
  4. The Top 5 Matching Host:IP Pair – lists the top 5 pairs and the number of occurrences.

Domain

This tool is used to investigate users and IPs that have visited a specified domain name to identify threats.

At the top of the page there are some filters which can be used to narrow down the search after entering a domain into the search bar on the left. The date range can be selected in a two ways, the first is by selecting one of the following the options “Last Day”, “Last Week”, “Last Month”, “3 Months”, “6 Months” and “12 Months” and the second is picking specific start and end dates using the datetime picker on the far right – if no range is specified the default “Last Month” is used.

The “Proxy Logs” section on the left will then be populated with the following three tables:

  • Top 10 NTIDs – Lists the count of records of the specified domain grouped by NTID.
    1. Clicking on the NTID redirects to the NTID tab, listing the user information filtering by clicked NTID.
    2. Clicking on the total count redirects to ADXsearch, listing data from Proxy Logs and filtering by clicked NTID and date. Results returned can be narrowed down further by entering the domain in the search bar at the top right of the table.
  • Top 10 Source IPs – Lists the count of records of the specified domain grouped by Source IP.
    1. Clicking on the Source IP Address redirects to the IP tab, filtering by clicked IP.
    2. Clicking on the total count redirected to ADXsearch, listing the data from Proxy Logs and filtering by clicked Source IP and date. Results returned can be narrowed down further by entering the domain in the search bar at the top right of the table.
  • Top 10 Destination IPs – Lists the count of records of the specified domain grouped by Destination IP.
    1. Clicking on the Destination IP Address redirects to the IP tab, filtering by clicked IP.
    2. Clicking on the total count redirects to ADXsearch, listing the data from Proxy Logs and filtering by clicked Destination IP and date. Results returned can be narrowed down further by entering the domain in the search bar at the top right of the table.

The “Cyber Threat Detected” section on the right shows a list of the relevant cyber threats previously detected by querying IOC data using the domain specified, it returns all rows of type “HOST”.

The table displays the following for each threat:

  • Priority – either “High” or “Very High”, only incidents given a score of 80/100 or higher are brough into ThreatConnect, with very few and exceptional exceptions (for example if a threating domain specifically targets BP).
  • Indicator – what specifically was the threat (what specific domain).
  • Owner – where the threat originated from, so if a User needs to go back to ThreatConnect and try to track down the source they can follow the TC Link and search the data source more thoroughly.
  • TC Link – a URL link that takes the User directly to the data source. This does require previous access permissions set by BP and cannot be documented here.
  • ArcSight RT – the date that the threat first touched based within ArcSight.

Methods to further investigate a threat:

  1. Clicking the total count above the table on the left redirects to the IOC tab, filtering by the specified domain and data range. Additionally, when in the IOC tab, the following default filters will be set “Action Taken” = false and “No Matches”.
  2. Clicking on the TC Link redirects to the data source in ThreatConnect.

IOC

The Incidents of Concern (IOC) tool is used to identify threats and to make historical searches of those threats with BP. This tool provides an intuitive and powerful interface that queries date using ADXsearch and allows for filtering and extensive analysis of threats by navigating to other useful tabs in SOCtopus to investigate specific details. For example, this platform can be used to find out whether BP NTIDs or IPs have been in any Domain before a cyber-attack.

  1. When you first click of the IOC tab, the ThreatConnect IOCs is populated with all records with the filtered Actioned = false, sorted by Last Updated descending.
  2. To refine the records, there are some filters just above the table including searching for a term and selecting date range. A priority level can also be selected from All, High and Very High; a type can be selected between All, Address, Email Address, File and Host; finally the search can be narrowed down by owner including All, BP, iSIGHT – FireEye iSIGHT Critical Infrastructure and iSIGHT – FireEye iSIGHT Cyber Espionage.
    Each record has the following 15 columns:
    • Date Added – the date the threat was added to IOC and is assigned by ThreatConnect.
    • Last Update – also assigned by ThreatConnect.
    • Priority – either “High” or “Very High”, only incidents given a score of 80/100 or higher are brough into ThreatConnect, with very few and exceptional exceptions (for example if a threating domain specifically targets BP).
    • Type – what type of threat it was (Address, Email Address, File or Host).
    • Indicator – what specifically was the threat (what specific Address, Email Address, File or Host triggered the response). Depending on what type of Indicator it is, depends on what fields is being searched within the log sources of data. So, for example, if it is an IP indicator or a network-based Indicator, the IOC will be looking through DNS logs, Proxy log, anything that is network based that may potentially have that indicator.
    • Confidence – the level of confidence that this is a malicious indicator, set by the vendor. Anything below 80 is deemed to be a low or medium threat and will be very unlikely to appear in IOC (for example, if it was extremely targeting intelligence to directly related to oil and gas, or specifically mentioning BP itself). Medium to Low indicators are routinely investigated by ArcSight in 24-hour periods.
    • Score – the threat score of the indicator, how likely it is to be malicious, also set by the vendor. Anything below a 3.0 score is deemed to be low or medium threat and will be very unlikely to appear in IOC (and like with Confidence, the exceptions will be for example, if it was extremely targeting intelligence directly related to oil and gas, or specifically mentioning BP itself). Both the Confidence and Score levels are ThreatConnect values, and are both used to determine whether something is a high, medium or low priority for BP.
    • Owner – where the threat originated from, so if a User needs to go back to ThreatConnect and try to track down the source they can follow the TC Link and search the data source more thoroughly.
    • ID
    • Hit Source
    • Actioned – whether the SOC team have investigated the threat and deemed it safe.
    • Actioned By – the individual within the SOC team who has completed the investigation, represented in the form of the user ID rather than a full name.
    • Actioned Time – when specifically, the threat was marked as actioned.
    • TC Link – a URL link that takes the User directly to the data source. This does require previous access permissions set by BP and cannot be documented here.
    • ArcSight RT – the date that the threat first touched based within ArcSight.
    • HasMatches
    • ADX RT – the date that the threat first touched base within ADX. The reason both ArcSight RT and ADX RT are shown is to show the drastic discrepancies of time between receipt of data between both platforms.

  3. To get further details of a particular record, click on the row.
  4. The top 100 lines of matching results is fetched and is returned in the “Match Results” table, again sorted, sorted by date descending. In the results there are the following columns: Date Time, Domain, NTID, Source IP, Destination IP, adHostname and Feed (Data Source). At the top right of the table is an “Export” button which downloads the results in a CSV file when clicked.
  5. Each of the returned results have the following 7 columns: Time, Domain, NTID/UPN, Source IP, Destination IP, Host Name and Datasource which all related to the triggered alert. The NTID/UPN value for a record can be clicked to be redirected to the NTID tab in SOCtopus which loads the user’s details. These matching records can be exported for further analysis.
  6. Once a record has been investigated and dealt with, click the “Take Action” at the top right of the “ThreatConnect IOCs” table. This will change the row to being action so will no longer be visible in the table unless you switch the “Action Taken” toggle to true. Screenshot highlighting the “Take Action” button and the “Action Taken” toggle.
  7. When in the “Action Taken” is set to true table view, any record here can be sent back to be review by removing the action - this places it back so it can be investigated further.

Once a search returns, the top “WhoIs Data” section is populated with details of the IP address.

The next section on the left is “Source Fire” which lists the count of records of the IP address grouped by Direction, Alert Category and Rule Name. Clicking on “Total” redirects to ADXsearch, listing data from Source Fire and filtering by clicked Alert Category and Rule Name of the row, IP (Source or Destination IP, depending on the Direction) and date.

To the right is the “Cyber Threat Detected” section which lists the relevant cyber threats previously detected by querying IOC data using the IP address specified.

The table displays the following for each threat:

  • Priority – either “High” or “Very High”, only incidents given a score of 80/100 or higher are brough into ThreatConnect, with very few and exceptional exceptions (for example if a threating domain specifically targets BP).
  • Indicator – what specifically was the threat (what specific domain).
  • Owner – where the threat originated from, so if a User needs to go back to ThreatConnect and try to track down the source they can follow the TC Link and search the data source more thoroughly.
  • TC Link – a URL link that takes the User directly to the data source. This does require previous access permissions set by BP and cannot be documented here.
  • ArcSight RT – the date that the threat first touched based within ArcSight.

Methods to further investigate a threat:

  1. Clicking the total count above the table on the left redirects to the IOC tab, filtering by the specified IP address and data range. Additionally, when in the IOC tab, the following default filters will be set “Action Taken” = false and “No Matches”.
  2. Clicking on the TC Link redirects to the data source in ThreatConnect.

Navigating to the DNS/Proxy tab displays the following three lists of data:

  1. The “NTID” section Lists the count of records filtered by the IP address and grouped by NTID.
    1. Clicking on an NTID redirects to the NTID tab, listing the user information filtering by clicked NTID.
    2. Clicking on the total count redirects to ADXsearch, listing the data from Proxy Logs and filtering by specified IP address and date. Results returned can be narrowed down further by entering the NTID in the search bar at the top right of the table.
  2. The “Domains Source” section lists the count of records filtered by the IP address and grouped by Domain reached as Source IP.
    1. Clicking on the Domain redirects to the Domain tab, filtering by the clicked Domain.
    2. Clicking the total count redirects to ADXsearch, listing the data from Proxy Logs and filtering by specified IP (filtered by Source IP) and date. Results returned can be narrowed down further by entering the domain in the search bar at the top right of the table.
  3. The “Domains Destination” section lists the count of records filtered by the IP address and grouped by Domain reached as Destination IP.
    1. Clicking on the Domain redirects to the Domain tab, filtering by the clicked Domain.
    2. Clicking the total count redirects to ADXsearch, listing the data from Proxy Logs and filtering by specified IP (filtered by Destination IP) and date. Results returned can be narrowed down further by entering the domain in the search bar at the top right of the table.

Host

This tool is used to investigate data related to a search host name, such as IP for that machine, who the machine belongs to and if the machine is linked to any IDS alerts.

At the top of the page there are some filters which can be used to narrow down the search after entering a host name into the search bar on the left. The date range can be selected in a two ways, the first is by selecting one of the following the options “Last Day”, “Last Week”, “Last Month”, “3 Months”, “6 Months” and “12 Months” and the second is picking specific start and end dates using the datetime picker on the far right – if no range is specified the default “Last Month” is used.

Once a search returns, the top "Host Details" section is populated with deatils of the specified host name. Clicking on the "Last" or "Previous" NTID displayed redirects to the NTID tab, filtering by the clicked NTID.

The next section has two tabs:

The first is the "Alerts" tab which displays a summary of alerts for the machine.

  1. Clicking on an alert redirect to ADXsearch, listing the data and filtering by the specified alert.

The second “IPs” tab displays a ranked summary of the most recent IPs with counts that the machine has had on the network.

  1. Clicking on the IP redirects to the IP tab, filtering by the clicked IP address.
  2. Clicking the total count redirects to ADXsearch, listing the data from Proxy Logs and filtering by specified IP address and date.
