FluentD Monitoring

FluentD Monitoring is a Power BI Workspace with reports and dashboards to monitor data processing from FluentD into SDS ADLS (Azure Data Leke Services).

It provides a consolidated point to see how data is landing and to identify ports and feeds that are not receiving data as frequently as expected.


(click on the image to open in a new tab)


Data Sources

FluentD Monitoring relies on debug and info logs received from the parser that sends data to SDS ADLS and also from a Databricks job that frequently scans the SDS datalake to detect the latest files received for each feed.

If a port number is not receiving data for 2 or more hours, an alert will be sent to the channel "DA1 CTU Analytics Integration and Automation" / "Power Bi Notifications" and to distribution emails G ITANDS DS SDS Ops Support (gitsdssdsopssupport@bp.com) and G IT&S DS Security Data Services (gitsdssecuritydataservices@bp.com).


Data Refresh

FluentD Model is refreshed every hour.
